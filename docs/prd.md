# PRD - WebCut

## 1. 프로젝트 개요

- **목적:** 브라우저에서 동작하는 웹 기반 동영상 편집기. 별도 설치 없이 브라우저만으로 영상 컷 편집, 트랜지션, 필터, 텍스트 오버레이, 내보내기까지 가능한 올인원 편집 도구를 제공한다.
- **해결하는 문제:** 전문 편집 소프트웨어(프리미어, 다빈치 등)는 비전문가에게 진입 장벽이 높다. 설치 없이 브라우저에서 직관적으로 영상을 편집할 수 있는 가벼운 도구가 필요하다.
- **기대 효과:** 비전문가도 몇 번의 클릭과 드래그만으로 영상을 편집하고 완성된 MP4 파일을 내보낼 수 있다.

## 2. 목표 사용자

- **주요 사용자:** 비전문가 크리에이터 — 유튜브 쇼츠, 인스타그램 릴스, 틱톡 등 짧은 영상을 만드는 일반 사용자. 간단한 편집이 필요한 직장인/학생.
- **사용자 특성:** 전문 편집 지식이 없으며, 직관적이고 단순한 인터페이스를 선호한다. 복잡한 기능보다 빠른 결과물 도출을 중시한다.

## 3. 핵심 기능

- [ ] **미디어 관리 (Media Pool):** 로컬 영상, 오디오, 이미지 파일 업로드 및 라이브러리 관리
- [ ] **타임라인 시퀀서 (Timeline):** 다중 트랙(비디오/오디오) 기반 드래그 앤 드롭 편집, 플레이헤드 및 스크러빙
- [ ] **기본 편집 도구:** 자르기(Split), 트림(Trim), 스냅핑(Snapping)
- [ ] **프리뷰 렌더링 (Canvas Player):** PixiJS 기반 실시간 프리뷰, 트랜스폼(위치/크기/회전) 제어
- [ ] **트랜지션 및 필터:** 페이드, 디졸브 트랜지션 + 밝기, 대비 등 기본 필터
- [ ] **텍스트 오버레이:** 영상 위에 텍스트 추가 및 스타일링
- [ ] **오디오 편집:** 별도 오디오 트랙 + 볼륨 조절
- [ ] **내보내기 (Export):** FFmpeg.wasm 기반 해상도 선택 가능 MP4 내보내기
- [ ] **Undo/Redo:** 편집 작업 실행 취소 및 다시 실행

## 4. 사용자 스토리

- 사용자로서, 로컬 영상/오디오/이미지 파일을 드래그 앤 드롭으로 업로드하고 싶다. 그래서 편집할 소스를 쉽게 준비할 수 있다.
- 사용자로서, 타임라인에 클립을 배치하고 순서를 변경하고 싶다. 그래서 원하는 흐름으로 영상을 구성할 수 있다.
- 사용자로서, 영상의 불필요한 부분을 잘라내고 싶다. 그래서 핵심 내용만 남길 수 있다.
- 사용자로서, 클립 사이에 트랜지션을 넣고 싶다. 그래서 장면 전환이 자연스러워진다.
- 사용자로서, 영상에 텍스트를 추가하고 싶다. 그래서 자막이나 제목을 넣을 수 있다.
- 사용자로서, 밝기/대비 등 기본 필터를 적용하고 싶다. 그래서 영상 품질을 보정할 수 있다.
- 사용자로서, 실시간 프리뷰로 편집 결과를 확인하고 싶다. 그래서 내보내기 전에 결과물을 미리 볼 수 있다.
- 사용자로서, 편집 실수를 Undo/Redo로 되돌리고 싶다. 그래서 안심하고 편집할 수 있다.
- 사용자로서, 원하는 해상도로 MP4 파일을 내보내고 싶다. 그래서 완성된 영상을 다운로드할 수 있다.

## 5. 기술 스택

- **프론트엔드:** Vite + React + TypeScript
- **상태 관리:** Zustand
- **렌더링 엔진:** PixiJS (Canvas/WebGL 기반 프리뷰)
- **인코딩:** FFmpeg.wasm (브라우저 내 동영상 인코딩)
- **UI 컴포넌트:** shadcn/ui (Radix + Tailwind)
- **CSS:** Tailwind CSS
- **드래그 앤 드롭:** dnd-kit
- **린트/포매팅:** Biome
- **테스트:** Vitest
- **배포:** Vercel
- **백엔드:** 없음 (순수 클라이언트 앱, 모든 처리를 브라우저에서 수행)

## 6. 비기능 요구사항

### 보안
- 모든 미디어 파일은 클라이언트 측에서만 처리되며, 외부 서버로 전송하지 않는다.
- 사용자 입력(텍스트 오버레이 등)은 XSS 방지를 위해 새니타이즈한다.

### 접근성
- 키보드 단축키 지원 (재생/정지, 자르기, Undo/Redo 등)
- 시맨틱 HTML 및 ARIA 속성 적용
- 색상 대비 비율 4.5:1 이상 유지 (다크 테마)

### 성능
- 대용량 미디어 파일은 `URL.createObjectURL()`로 참조하여 메모리 부하를 최소화한다.
- FFmpeg.wasm 인코딩은 Web Worker에서 처리하여 UI 블로킹을 방지한다.
- PixiJS WebGL 렌더링으로 부드러운 프리뷰 재생을 보장한다.
- 60fps 타겟의 타임라인 스크러빙 응답성을 유지한다.

### 호환성
- 데스크톱 우선 + 태블릿 지원
- Chrome, Firefox, Safari, Edge 최신 버전 지원
- SharedArrayBuffer 지원이 필요한 FFmpeg.wasm을 위한 COOP/COEP 헤더 설정
