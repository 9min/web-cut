이 프로젝트는 크게 **UI/상태 관리 영역**, **비디오 렌더링 엔진**, 그리고 **인코딩/내보내기 영역**으로 나눌 수 있습니다. 프론트엔드 아키텍처 관점에서 개발 마일스톤을 세우기 좋도록 구성했습니다.

---

### Phase 1: 미디어 관리 및 로드 (Media Pool)
사용자의 로컬 환경에서 브라우저로 대용량 미디어 파일을 가져오고 관리하는 기능입니다.

* **로컬 파일 업로드 및 파싱:** `File API`를 사용하여 사용자의 로컬 영상, 오디오, 이미지 파일을 브라우저로 불러옵니다. 대용량 파일의 브라우저 메모리 부하를 막기 위해 `URL.createObjectURL()`을 활용하여 참조 링크만 생성합니다.
* **미디어 메타데이터 추출:** 불러온 비디오의 해상도, 프레임 레이트(fps), 전체 길이(Duration) 정보를 추출하여 상태 저장소에 보관합니다.
* **미디어 라이브러리 UI:** 불러온 에셋들을 목록 형태로 보여주고, 드래그 앤 드롭으로 타임라인에 올릴 수 있는 준비 상태를 만듭니다.

### Phase 2: 타임라인 및 상태 관리 (Timeline Sequencer)
편집기의 뼈대이자, 프론트엔드 상태 관리가 가장 복잡해지는 핵심 영역입니다. 

* **트랙(Track) 시스템 구현:** 비디오 트랙과 오디오 트랙을 분리하여 생성, 삭제, 순서 변경이 가능한 다중 레이어 구조를 만듭니다.
* **클립(Clip) 배치 및 이동:** 미디어 라이브러리에서 타임라인으로 클립을 드래그 앤 드롭으로 배치합니다. x축은 '시간', y축은 '레이어 우선순위'를 의미합니다.
* **정밀한 상태 관리 (State Management):** 각 클립의 고유 ID, 소스 파일 참조, 타임라인 상의 시작/종료 시간(in/out point), 트림(Trim)된 실제 재생 구간 정보를 객체 배열 형태로 정교하게 관리해야 합니다.
* **플레이헤드 (Playhead) 및 스크러빙:** 현재 재생 위치를 나타내는 인디케이터를 구현합니다. 마우스로 드래그할 때(Scrubbing) 렌더러와 즉각적으로 동기화되어 화면이 바뀌어야 합니다.

### Phase 3: 기본 편집 도구 (Core Editing Tools)
실제 컷 편집을 수행하는 액션 기능들입니다.

* **자르기 (Split / Razor Tool):** 특정 시간(플레이헤드 위치)을 기준으로 하나의 클립을 두 개의 독립적인 클립 데이터로 분할합니다.
* **길이 조절 (Trim):** 클립의 좌우 끝단을 마우스로 드래그하여 영상의 시작점과 끝점을 줄이거나 늘립니다.
* **스냅핑 (Snapping):** 클립을 이동할 때 다른 클립의 시작/끝점이나 플레이헤드 근처에 가면 자석처럼 달라붙게 하여 정밀한 편집을 돕습니다.

### Phase 4: 프리뷰 렌더링 엔진 (Canvas Player)
타임라인의 상태 데이터를 바탕으로 현재 시간에 해당하는 화면을 브라우저에 그려주는 역할입니다. 

* **Canvas 기반 렌더링:** HTML5 `<video>` 태그 여러 개를 겹치는 방식은 한계가 명확합니다. 성능 최적화와 향후 트랜지션, 필터 확장을 위해 `<canvas>`를 사용해야 합니다. WebGL을 직접 다루거나 PixiJS 같은 고성능 2D/WebGL 렌더링 라이브러리를 도입하여 비디오 프레임을 텍스처로 렌더링하는 방식이 효과적입니다.
* **시간 동기화 (Time Sync):** `requestAnimationFrame`을 활용하여 플레이헤드의 현재 시간(currentTime)과 타임라인 상에 위치한 클립들의 데이터를 계산해, 해당 시간에 보여야 할 비디오 프레임만 정확히 캔버스에 그립니다.
* **기본 트랜스폼 제어:** 화면 내에서 영상 클립의 위치(Position), 크기(Scale), 회전(Rotation)을 조절할 수 있는 기능을 구현합니다.

### Phase 5: 렌더링 및 내보내기 (Export)
타임라인에서 편집된 결과물을 하나의 실제 동영상 파일(.mp4 등)로 합쳐서 다운로드하는 최종 단계입니다.

* **FFmpeg.wasm 연동:** 브라우저 환경에서 FFmpeg를 구동합니다.
* **편집 데이터 변환:** 타임라인에 저장된 자르기, 배치, 트랜스폼 상태 데이터를 FFmpeg가 이해할 수 있는 복잡한 명령어(Command line arguments) 또는 Filter Graph 문법으로 자동 변환하는 로직이 필요합니다.
* **인코딩 프로세스 및 진행률:** FFmpeg가 백그라운드(Web Worker 추천)에서 파일들을 잘라 붙이고 인코딩하는 동안의 진행 상태(Progress)를 UI로 표시하고, 완료 시 `Blob` 데이터를 생성해 파일로 다운로드합니다.
